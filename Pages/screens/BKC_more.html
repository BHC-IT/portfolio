BKC was build for monkey money. The goal was to give french local money association the ability to digitalise their monney.

The association owner needed to run the chaine nodes on cheap machines, and could register huge spike on specific days without the possibility to scale. We thinked BKC just for that, beeing as low consuming as possible will handling large amount of transaction at the same time.

Also, association had only a few Desktop to spare to serve as node for the chain. If one nodes crashed, it could mean a huge loss in capacity for them. The software was made to handle any bad that could happen, as long as the PC is running.

Build from scratch in C++ with an actor model, BKC is heavily optimised to run on multi-core CPU and can handle high load efficiently.

Knowing the chain would be run in private environnement, we used a lazy approach to transaction, allowing thousands of request to be handled by the chain.

Using strict concurrency model such as actor and pi calculus, the process can recover from anything and will, at worse, delay treatment
